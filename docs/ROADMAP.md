# Suggested build order

## Foundation

- Replace placeholder links and biography data.
- Decide the permanent domain and GitHub Pages base path.
- Add real project, performance, chess, and pickleball entries.
- Establish typography and photography direction.

## First signature interaction

- Build one homepage interaction that connects the five disciplines.
- Keep the current static hero as the fallback.
- Measure load cost and reduced-motion behavior.

## Domain experiences

- Clarinet: recording player plus score or fingering visualization.
- Piano: repertoire archive plus keyboard or sight-reading visualization.
- Chess: PGN viewer and annotated critical positions.
- Pickleball: court trajectory or training data visualization.
- Code: live demos and architecture diagrams.

## Seasonal engine

- Define a small renderer interface.
- Build one high-quality season first.
- Add cached static layers and active growth layers.
- Add low-power and studio modes before adding the remaining seasons.

## Observatory

- Choose an analytics provider.
- Replace `scripts/generate-stats.mjs` with a provider adapter.
- Add custom interaction aggregates.
- Add Lighthouse or bundle metrics during CI.
- Publish only anonymous aggregate data.
